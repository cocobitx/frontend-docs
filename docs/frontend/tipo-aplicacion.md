La distinción entre Sitios Web Estáticos y Aplicaciones Web (Web Apps) reside en el nivel de interactividad, el procesamiento de datos y la arquitectura técnica subyacente. Esta clasificación determina cómo se entrega el contenido al usuario y qué recursos de servidor son necesarios.

Cada tecnología está orientada a un tipo específico de la aplicación. Su elección depende de los requerimientos funcionales, de rendimiento y de plataforma, ya que cada una ofrece ventajas particulares según el contexto de uso y el tipo de solución a desarrollar.

![Imagen aplicaciones](/aplicaciones.png)

- Public Web Applications: Es un software interactivo que se ejecuta en el navegador, capaz de procesar datos en tiempo real y personalizar la experiencia del usuario de forma dinámica, es importante el manejo del SEO
- Lading Page: Conjunto de documentos almacenados en un servidor que se entregan al navegador del usuario sin sufrir modificaciones en el servidor. Su contenido es informativo y constante.
- Pub/Piv Web Applications:  Este tipo de software es hibrido, por lo tanto no se prioriza SEO y se prioriza flexiblidad y funcionamiento, tiene integraciones para compilar la web en aplicaciones de escritorios y moviles.
